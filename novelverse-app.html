Tugas 1a — Flowchart

Flowchart CRUD NovelVerse

Diagram alir yang menggambarkan proses Tambah, Tampil, Ubah, dan Hapus data novel menggunakan simbol flowchart standar ISO 5807.

1️⃣ Tambah Data Novel
Alur proses ketika user menambahkan novel baru ke sistem
START
INPUT: Judul, Genre, Status, Sinopsis
Input Kosong?
YA → Error TIDAK →
Buat objek novel baru
{ id, title, genre, status, synopsis, date }
Ambil array dari LocalStorage
novels = getFromStorage()
Push novel baru ke array
novels.push(newNovel)
Simpan kembali ke LocalStorage
localStorage.setItem("novels", JSON)
OUTPUT: Render ulang daftar novel
OUTPUT: Tampilkan Toast Notifikasi ✅
END
2️⃣ Tampilkan Data Novel
Alur looping untuk merender semua novel dari LocalStorage ke DOM
START
Ambil data dari LocalStorage
JSON.parse(localStorage.getItem)
Data Kosong?
YA → Empty State TIDAK →
Inisialisasi: i = 0
i < novels.length?
TIDAK → Stop Loop YA →
Buat elemen card HTML
createNovelCard(novels[i])
Append card ke DOM
novelList.innerHTML += card
i++ → kembali ke cek kondisi
OUTPUT: Semua novel ter-render di halaman
END
3️⃣ Ubah / Edit Data Novel
Alur proses edit data — dari klik tombol edit sampai data tersimpan
START
INPUT: User klik tombol "Edit" pada novel
Cari novel berdasarkan ID
novels.find(n => n.id === id)
Isi form modal dengan data lama
pre-populate input fields
OUTPUT: Modal Edit terbuka
INPUT: User mengubah data & klik Simpan
Input Valid?
TIDAK → Tampil Error YA →
Temukan index novel di array
findIndex(n => n.id === id)
Update properti novel
novels[idx] = { ...old, ...new }
Simpan array ke LocalStorage
OUTPUT: Re-render list + toast sukses
END
4️⃣ Hapus Data Novel
Alur konfirmasi sebelum menghapus novel dari sistem
START
INPUT: User klik tombol "Hapus"
OUTPUT: Modal konfirmasi tampil
User Konfirmasi Hapus?
TIDAK → Tutup modal YA →
Filter array: hapus novel dengan ID
novels = novels.filter(n => n.id !== id)
Simpan array baru ke LocalStorage
Hapus chapter yang terkait dengan novel
chapters = chapters.filter(c => c.novelId !== id)
OUTPUT: Re-render list + toast berhasil
END
Penjelasan Alur Program

Ringkasan Alur

📥
Tambah (CREATE)
User mengisi form → validasi input (IF kosong → error) → buat objek novel dengan ID unik → push ke array → simpan ke LocalStorage → render ulang DOM.
📋
Tampil (READ)
Baca array dari LocalStorage → jika kosong tampilkan empty state → LOOP forEach setiap novel → buat HTML card → append ke container. Filter realtime saat user mengetik.
✏️
Ubah (UPDATE)
Klik Edit → cari novel by ID → isi modal → user ubah data → validasi → findIndex → update objek → simpan ke LocalStorage → re-render.
🗑️
Hapus (DELETE)
Klik Hapus → konfirmasi modal → IF konfirmasi YES → filter array (remove by ID) → hapus chapter terkait → simpan → re-render.
Tugas 1b + 2 + 3 — Pseudocode & Struktur Kontrol

Pseudocode NovelVerse

Pseudocode lengkap yang mencakup: Sequence, If/Else, Looping, LocalStorage, DOM Manipulation, dan notifikasi interaktif.

PROCEDURE AddNovel()
Menambah novel baru ke LocalStorage dengan validasi input
// ═══ SEQUENCE: Urutan langkah yang harus dijalankan ═══ PROCEDURE AddNovel(): // SEQUENCE 1: Ambil nilai dari input form title ← GET_VALUE("inp-title").trim() genre ← GET_VALUE("inp-genre") status ← GET_VALUE("inp-status") synopsis ← GET_VALUE("inp-synopsis").trim() target ← GET_VALUE("inp-target") OR 0 // SEQUENCE 2 + IF/ELSE: Validasi input satu per satu isValid ← TRUE IF title = "" THEN SHOW_ERROR("err-title") // Tampilkan pesan error judul isValid ← FALSE END IF IF genre = "" THEN SHOW_ERROR("err-genre") // Tampilkan pesan error genre isValid ← FALSE END IF IF synopsis = "" THEN SHOW_ERROR("err-synopsis") // Tampilkan pesan error sinopsis isValid ← FALSE END IF IF isValid = FALSE THEN SHOW_TOAST("Mohon lengkapi semua field!", "error") RETURN // Hentikan eksekusi jika ada field kosong END IF // SEQUENCE 3: Buat objek novel baru newNovel ← { id: GENERATE_ID(), // ID unik timestamp title: title, genre: genre, status: status, synopsis: synopsis, wordCount: 0, target: parseInt(target), createdAt: NOW() } // SEQUENCE 4: Ambil data dari LocalStorage, tambahkan, simpan novels ← LOAD_FROM_STORAGE("nv_novels") novels.push(newNovel) SAVE_TO_STORAGE("nv_novels", novels) // SEQUENCE 5: Reset form + re-render + notifikasi RESET_FORM() RENDER_NOVEL_LIST() UPDATE_STATS() SHOW_TOAST("Novel berhasil ditambahkan! 🎉", "success") END PROCEDURE
PROCEDURE RenderNovelList()
Menampilkan daftar novel menggunakan DOM Manipulation + LOOPING
// ═══ LOOPING: forEach untuk iterasi semua novel ═══ PROCEDURE RenderNovelList(searchQuery = "", filterGenre = ""): container ← GET_ELEMENT("novel-list") novels ← LOAD_FROM_STORAGE("nv_novels") // IF/ELSE: Cek apakah data kosong → tampilkan empty state IF novels.length = 0 THEN container.innerHTML ← RENDER_EMPTY_STATE() RETURN END IF // LOOPING 1: Filter berdasarkan search query (jika ada) IF searchQuery ≠ "" OR filterGenre ≠ "" THEN filtered ← [] FOR EACH novel IN novels DO matchSearch ← novel.title.includes(searchQuery) OR novel.genre.includes(searchQuery) matchGenre ← (filterGenre = "") OR (novel.genre = filterGenre) IF matchSearch AND matchGenre THEN filtered.push(novel) END IF END FOR novels ← filtered END IF // LOOPING 2: Render setiap novel ke DOM (DOM Manipulation) html ← "" FOR EACH novel IN novels DO // IF/ELSE: Tentukan warna badge status IF novel.status = "Completed" THEN badgeClass ← "badge-success" ELSE IF novel.status = "Ongoing" THEN badgeClass ← "badge-info" ELSE IF novel.status = "Hiatus" THEN badgeClass ← "badge-danger" ELSE badgeClass ← "badge-gold" END IF html ← html + BUILD_CARD_HTML(novel, badgeClass) END FOR container.innerHTML ← html // DOM Manipulation: update tampilan END PROCEDURE
PROCEDURE EditNovel(id) + SaveEditNovel()
Dua prosedur yang bekerja berurutan untuk update data
// ═══ Prosedur 1: Buka modal edit & pre-populate data ═══ PROCEDURE EditNovel(novelId): novels ← LOAD_FROM_STORAGE("nv_novels") // LOOPING: Cari novel berdasarkan ID novel ← novels.find(n ⟹ n.id = novelId) // IF/ELSE: Pastikan novel ditemukan IF novel = NULL THEN SHOW_TOAST("Novel tidak ditemukan!", "error") RETURN END IF // SEQUENCE: Isi form modal dengan data lama SET_VALUE("edit-novel-id", novel.id) SET_VALUE("edit-title", novel.title) SET_VALUE("edit-genre", novel.genre) SET_VALUE("edit-status", novel.status) SET_VALUE("edit-synopsis", novel.synopsis) OPEN_MODAL("modal-edit") END PROCEDURE // ═══ Prosedur 2: Simpan hasil edit ═══ PROCEDURE SaveEditNovel(): id ← GET_VALUE("edit-novel-id") title ← GET_VALUE("edit-title").trim() genre ← GET_VALUE("edit-genre") synopsis ← GET_VALUE("edit-synopsis").trim() // IF/ELSE: Validasi input IF title = "" OR genre = "" OR synopsis = "" THEN SHOW_TOAST("Semua field harus diisi!", "error") RETURN END IF novels ← LOAD_FROM_STORAGE("nv_novels") idx ← novels.findIndex(n ⟹ n.id = id) // SEQUENCE: Update properti novel yang ditemukan novels[idx].title ← title novels[idx].genre ← genre novels[idx].status ← status novels[idx].synopsis ← synopsis novels[idx].updatedAt ← NOW() SAVE_TO_STORAGE("nv_novels", novels) CLOSE_MODAL("modal-edit") RENDER_NOVEL_LIST() SHOW_TOAST("Novel berhasil diperbarui! ✅", "success") END PROCEDURE
PROCEDURE DeleteNovel(id)
Menghapus novel beserta chapter-chapter terkait dari storage
// ═══ Prosedur Hapus dengan konfirmasi modal ═══ PROCEDURE OpenDeleteConfirm(novelId): novel ← FIND_NOVEL(novelId) SET_VALUE("delete-novel-id", novelId) SET_TEXT("delete-novel-title", novel.title) OPEN_MODAL("modal-delete") END PROCEDURE PROCEDURE ConfirmDeleteNovel(): id ← GET_VALUE("delete-novel-id") novels ← LOAD_FROM_STORAGE("nv_novels") // LOOPING via filter: Hapus novel yang id-nya cocok novels ← novels.filter(n ⟹ n.id ≠ id) SAVE_TO_STORAGE("nv_novels", novels) // Hapus juga semua chapter yang terkait dengan novel ini chapters ← LOAD_FROM_STORAGE("nv_chapters") chapters ← chapters.filter(c ⟹ c.novelId ≠ id) SAVE_TO_STORAGE("nv_chapters", chapters) CLOSE_MODAL("modal-delete") RENDER_NOVEL_LIST() UPDATE_STATS() SHOW_TOAST("Novel telah dihapus.", "warning") END PROCEDURE
PROCEDURE Storage Helpers
Fungsi utilitas untuk operasi LocalStorage
// ═══ Helper: Load dari LocalStorage ═══ FUNCTION loadFromStorage(key): raw ← localStorage.getItem(key) IF raw = NULL THEN RETURN [] // Kembalikan array kosong jika belum ada data ELSE RETURN JSON.parse(raw) END IF END FUNCTION // ═══ Helper: Simpan ke LocalStorage ═══ FUNCTION saveToStorage(key, data): localStorage.setItem(key, JSON.stringify(data)) END FUNCTION // ═══ Helper: Notifikasi Toast ═══ FUNCTION showToast(message, type): icon ← "ℹ️" IF type = "success" THEN icon ← "✅" ELSE IF type = "error" THEN icon ← "❌" ELSE IF type = "warning" THEN icon ← "⚠️" END IF toastEl ← CREATE_ELEMENT("div") toastEl.className ← "toast " + type toastEl.innerHTML ← icon + " " + message APPEND("toast-container", toastEl) DELAY(3000ms) THEN REMOVE(toastEl) // Auto-dismiss setelah 3 detik END FUNCTION
